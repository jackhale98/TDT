// Features

Built for Engineering Teams

Everything you need to manage complex product development in plain text.

```

Plain-Text YAML

Human-readable files that diff beautifully in git. No proprietary formats, no vendor lock-in. Your data stays yours forever.

Schema Validation

JSON Schema validation with beautiful error messages. Line numbers, context, and actionable suggestions for every issue.

Full Traceability

Link requirements to tests, risks to mitigations. Generate traceability matrices and find orphaned entities instantly.

FMEA Risk Management

Built-in Failure Mode and Effects Analysis. Track severity, occurrence, detection ratings. Auto-calculate RPN and risk levels.

Short ID Aliases

Use REQ@1, RISK@2 instead of 26-character ULIDs. Persistent, cross-entity linking that just works.

Tolerance Analysis

Features, mates, and stackups with worst-case, RSS, and Monte Carlo analysis. Built for mechanical engineering.

```
// Advanced Workflows

More Than Requirements

TDT powers your entire product lifecycle with specialized engineering tools.

Tolerance Stackup Analysis

Define features on components, create mates, and run worst-case, RSS, and Monte Carlo analysis on tolerance chains.

# Create features on components
$ tdt feat new -c CMP@1 -t hole --title "Bore"
✓ Created feature FEAT@1
   Parent: CMP-01KC5W... | Type: hole | Bore

# Run Monte Carlo analysis
$ tdt tol analyze TOL@1 --iterations 10000
⚙ Analyzing stackup TOL@1...
✓ Analysis complete

   Worst-Case: 0.87 to 1.18 pass
   RSS: Cpk 4.56 | Yield 99.99%
   Monte Carlo: Yield 100.00%

FMEA Risk Management

Create design and process risks with severity, occurrence, and detection ratings. Auto-calculate RPN and track mitigations.

# Create a risk with FMEA ratings
$ tdt risk new --title "Motor Overheating" -t design \
    --severity 8 --occurrence 4 --detection 5
✓ Created risk RISK@1
   RPN: 160 (high)

# View risks sorted by RPN
$ tdt risk list --by-rpn
SHORT   TYPE     TITLE               LEVEL   RPN
RISK@1  design   Motor Overheating   high    160
RISK@2  process  Seal Degradation    medium   72

Manufacturing Quality Loop

Define processes, control plans, work instructions, and track NCRs back to CAPAs for continuous improvement.

# Create a manufacturing process
$ tdt proc new -t "CNC Milling" -T machining
✓ Created process PROC@1
   Type: machining | CNC Milling

# Add SPC control point
$ tdt ctrl new -t "Bore SPC" -T spc -p PROC@1 --critical
✓ Created control CTRL@1
   Type: spc | Bore SPC [CTQ]

# Log NCR and create corrective action
$ tdt ncr new -t "Bore OOT" -S major
✓ Created NCR NCR@1
   internal | major | Bore OOT

Full Traceability

Link requirements to tests, risks to mitigations. Generate coverage reports and find orphaned entities.

# Link requirement to test
$ tdt link add REQ@1 TEST@1 -t verified_by
✓ Added link: REQ-01KC5W... --[verified_by]--> TEST-01KC5W...

# Check verification coverage
$ tdt trace coverage
Coverage: 92% (22/24 requirements verified)

// Getting Started

Up and Running in Minutes

Install TDT with a single command and initialize your first project. Works on macOS, Linux, and Windows.

```
1

Install with Cargo

Requires Rust toolchain installed

2

Initialize a Project

Creates directory structure and config

3

Start Creating

Add requirements, risks, tests, and more

Terminal 
# Install from crates.io
```

$ cargo install tdt

# Or build from source
$ git clone https://github.com/jackhale98/TDT.git
$ cd tdt && cargo build –release

# Initialize your project
$ tdt init
✓ Initialized TDT project

# Create your first requirement
$ tdt req new -t input –title "Temperature Range"
✓ Created requirement REQ@1

# Validate everything
$ tdt validate
✓ All files valid
